UI主题自定义面板位于：用户设置 → 主题颜色 上方

使用说明

在自定义CSS中为属性添加标记注释，即可在上方面板中可视化修改图片、文字、颜色和尺寸。

注意：同一面板中标记名称不能重复！
标记格式
图片 /* ggg-img: 名称 */
/* ggg-img-user: 名称 */
/* ggg-img-char: 名称 */
放在 url('...') 的前面（同一行或上一行均可）
文字 /* ggg-text: 名称 */
/* ggg-text-user: 名称 */
/* ggg-text-char: 名称 */
放在 content: "..." 的前面（同一行或上一行均可）
颜色 /* ggg-color: 名称 */
/* ggg-color-user: 名称 */
/* ggg-color-char: 名称 */
放在颜色值或CSS变量声明的前面（同一行或上一行均可）
尺寸 /* ggg-dim: 名称 */
/* ggg-dim-user: 名称 */
/* ggg-dim-char: 名称 */
放在CSS数值属性（如 top、width、height 等）的上一行
使用示例
以下示例美化代码来自呱呱 
/* 图片标记 - 标记在上一行 */
#chat .mes .ch_name::before {
    /* ggg-img-char: 闪烁星星图 */
    background-image: url('https://i.postimg.cc/JzNHCLGK/1-1-1.png');
    background-size: auto 100%;
}

/* 图片标记 - 标记在同一行 */
#chat .mes .ch_name::after {
    background-image: /* ggg-img-user: 闪烁星星图 */url('https://i.postimg.cc/mk42wcSs/2-1-1.png');
}

/* 文字标记 */
.mes:not([is_user="true"])::before {
    /* ggg-text: 搜索框文字 */
    content: "crush约我吃日料但我海鲜过敏怎么办T T";
}

/* 颜色标记 - 标记在上一行（推荐） */
:root {
    /* ggg-color: 主题色 */
    --blur-tint-color: rgba(178, 152, 164, 1);
}

/* 颜色标记 - 标记在同一行 */
:root {
    --my-border: /* ggg-color-char: 角色边框色 */ #ff6b9d;
}

/* 尺寸标记 - 用于自定义位置和大小 */
.mes:not([is_user="true"]) .mesAvatarWrapper .mesIDDisplay::after {
    content: "";
    position: absolute;
    /* ggg-dim: 增大下移 */
    top: -45px;
    /* ggg-dim: 增大左移 */
    right: -30px;
    /* ggg-dim: 宽 */
    width: 70px;
    /* ggg-dim: 高 */
    height: 70px;
    /* ggg-img-char: 楼层背图 */
    background: url('https://i.postimg.cc/jqn88QPT/hui.png') no-repeat center;
    background-size: contain;
}

/* 歌词动画中的文字标记 */
@keyframes userLyrics {
    /* ggg-text-user: user歌词1 */
    0%, 14.6% { content: "You and I are a story"; }
    /* ggg-text-user: user歌词2 */
    14.7%, 30.3% { content: "That never gets told"; }
}
注入HTML
使用说明

每个条目可同时包含 CSS 和 HTML（任一可空），绑定当前主题，切换主题时自动切换。

  • CSS：注入为 <style> 标签到 <head>，可用酒馆主题变量（var(--SmartThemeBodyColor) 等）
  • HTML：插入到目标 DOM 位置，<script> 会被执行；推荐用内置 api 操作 DOM / 事件 / 定时器
  • 覆盖酒馆原样式加 !important
  • 加入 /* ggg-color: 名称 */ 标记 → 在「UI主题自定义」面板里可视化拾色修改

三个字段的含义：

  • 条目名称：起个你自己看得懂的名字，显示在下方列表里。
  • 目标选择器：仅 HTML 用。留空则插到 <body> 末尾；或填 #chat 等 CSS 选择器。
  • 插入位置：相对目标元素：内部末尾 / 内部开头 / 元素之后 / 元素之前。

JS 推荐写法：

  • 推荐通过 api 操作：api.query / api.queryAll / api.append / api.on / api.timeout / api.observe / api.resizeObserve / api.setStyle / api.setAttr
  • 需要卸载时执行的逻辑可写 cleanup(() => ...)
  • 直接写 document / window / 原生 setTimeout / MutationObserver 也会执行，但插件不会可靠跟踪这些副作用；想要主题切换后可卸载、可恢复，优先用 api

api 作用速查：

  • api.query(selector, root?)：查找单个节点
  • api.queryAll(selector, root?)：查找多个节点，返回数组
  • api.create(tag, props?)：创建元素，props 可写 text / html / className
  • api.append / api.prepend / api.before / api.after：插入节点
  • api.remove(node)：删除节点
  • api.setAttr(node, name, value)：设置属性，并记录旧值方便卸载恢复
  • api.setStyle(node, name, value)：设置内联样式，并记录旧值方便卸载恢复
  • api.on / api.off：绑定 / 解绑事件，插件会跟踪监听器
  • api.timeout / api.clearTimeout：定时器，卸载时会自动清理未结束的定时器
  • api.interval / api.clearInterval：循环定时器
  • api.raf / api.cancelRaf：动画帧回调
  • api.observe(target, options, callback)：创建 MutationObserver
  • api.resizeObserve(target, callback)：创建 ResizeObserver
  • api.cleanup(fn)：注册卸载时执行的清理函数
  • api.setGlobal(key, value)：写入 window 全局变量，并在卸载时恢复
  • api.state：当前脚本自己的临时状态对象
  • api.log(...args)：输出带前缀日志

## 代码注入方式
- 「CSS」：直接注入酒馆页面的 <style> 标签，写标准 CSS 即可
- 「HTML」：将任意 HTML / JS 插入页面 DOM，<script> 标签会被执行
- 同一条目可同时包含 CSS 和 HTML（任一可空），共享条目名称

## HTML 注入注意事项
- 覆盖酒馆原有样式需加 !important
- 悬浮元素用 position:fixed（不填目标选择器时插到 <body> 末尾）
- JS 脚本直接写 <script>...</script> 放入 HTML 框
- 优先使用插件内置 api，不要直接裸用 window / document 乱改页面
- 可用 api：
  - api.query(selector, root?)：查单个节点
  - api.queryAll(selector, root?)：查多个节点，返回数组
  - api.create(tag, props?)：创建元素；props 可写 text / html / className
  - api.append(parent, child)：追加到末尾
  - api.prepend(parent, child)：插到开头
  - api.before(target, node)：插到目标前
  - api.after(target, node)：插到目标后
  - api.remove(node)：删除节点
  - api.setAttr(node, name, value)：设置属性，并记录旧值
  - api.setStyle(node, name, value)：设置内联样式，并记录旧值
  - api.on(target, type, handler, options?)：绑定事件并跟踪
  - api.off(target, type, handler, options?)：解绑事件
  - api.timeout(fn, ms) / api.clearTimeout(id)：定时器
  - api.interval(fn, ms) / api.clearInterval(id)：循环定时器
  - api.raf(fn) / api.cancelRaf(id)：动画帧回调
  - api.observe(target, options, callback)：MutationObserver
  - api.resizeObserve(target, callback)：ResizeObserver
  - api.cleanup(fn)：注册卸载清理逻辑
  - api.setGlobal(key, value)：设置 window 全局变量，并在卸载时恢复
  - api.state：当前脚本自己的临时状态对象
  - api.log(...args)：输出带前缀日志
- 可使用 cleanup(() => { ... }) 注册卸载清理逻辑
  - 直接写 document/window/原生定时器/原生观察器也会执行，但插件不会可靠跟踪这些副作用；想要主题切换后可卸载、可恢复，优先用 api
  - 如无必要，不要发网络请求，不要改原型链，不要长期占用全局变量

全屏兼容性
移动端浏览器（Chrome、Edge 等）的全屏模式存在渲染缺陷：软键盘弹出时，弹窗内容可能变透明、浏览器自动填充栏可能飘到错误位置。 以下选项通过在聚焦时临时退出全屏来规避这些问题，关闭弹窗或取消聚焦后会自动恢复全屏。 代价是进出全屏时屏幕会短暂闪黑（浏览器自身的过渡动画，无法消除）。

自定义时间正则说明

自定义时间会在进入手机时从最新匹配楼层取锚点，之后按真实时间流逝；跨天会自动更新日期和星期。

占位符指正则捕获组序号，也就是第几个 (...)。可填数字，也可填命名组名。

示例

原始文本：

»»»2023-5-16|周一|12:16|晴|Chs.1 嗯|北京|北京欢迎你«««

正则：

»»»(\d{4}-\d{1,2}-\d{1,2})\|([^|]+)\|([^|]+)\|([^|]+)\|Chs\.(\d+)\s*([^|]+)\|([^|]+)\|([^«]+)«««
12023-5-16
日期占位符填 1
2周一
星期占位符填 2
312:16
时间占位符填 3
4晴
天气占位符填 4

因此这条格式应填写：日期 1、时间 3、星期 2、天气 4。如果日期填 1、时间填 2，会把 周一 当成时间，解析会失败。
